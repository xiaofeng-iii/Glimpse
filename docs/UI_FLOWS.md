# Glimpse 用户交互流程

## 1. 搜索流程

### 1.1 标准搜索（综合结果）

```
┌─────────────┐
│ 输入搜索词   │
└──────┬──────┘
       │ textChanged (300ms 防抖)
       ▼
┌─────────────┐
│ _do_search() │
└──────┬──────┘
       │ source_filter = "all"
       ▼
┌─────────────┐
│ SearchService│
│ .search()    │
└──────┬──────┘
       │ 同时执行：
       │ 1. SQLite FTS5 文本搜索
       │ 2. ChromaDB 向量搜索
       ▼
┌─────────────┐
│ RRF 融合排名 │
└──────┬──────┘
       │ 合并 + 去重
       ▼
┌─────────────┐
│ 标注来源标签 │
└──────┬──────┘
       │ 精确 / 语义
       ▼
┌─────────────┐
│ _update_memory_list()
│ 渲染 [精确][语义] 前缀
└─────────────┘
```

### 1.2 来源过滤搜索

用户通过分段按钮组（`SegmentedFilterControl`，三按钮互斥切换）选择不同来源时的行为差异：

| 选择 | `source_filter` | 执行方法 | 搜索结果 | 显示标签 |
|------|-----------------|----------|----------|----------|
| `综合结果` | `"all"` | `_search_hybrid()` | 文本+向量融合 | `[精确]` / `[语义]` / 两者 |
| `仅看精确` | `"ocr"` | `_search_text()` | 仅文本匹配 | `[精确]` |
| `仅看语义` | `"semantic"` | `_search_vector()` | 仅向量匹配 | `[语义]` |

**注意**: 切换按钮会立即触发 `_do_search()`，无需重新输入搜索词。`source_filter` 参数传递给 `search_service.search()`，在内部覆写 `_search_mode`。

## 2. 截图流程

### 2.1 单张截图

```
用户触发截图（按钮/快捷键）
       │
       ▼
隐藏主窗口
       │
       ▼
执行截图（250ms 延迟）
       │
       ▼
显示主窗口
       │
       ▼
发送信号 → MemoryService.create_memory_async()
       │
       ▼
状态栏显示进度："正在分析..."
       │
       ▼
完成回调 → 状态栏："记忆已保存 xxx"
       │
       ▼
发射 memory_saved 信号 → 刷新列表
```

### 2.2 集群截图

```
启用集群模式（SettingsDialog）
       │
       ▼
第一次截图
       │
       ▼
进入 COLLECTING 状态
       │
       ▼
显示"立即提交"和"取消"按钮
       │
       ├─ 用户继续截图 → 追加到缓冲区
       │   状态栏："已收集 N/M 张截图"
       │
       ├─ 用户点击"立即提交" → 分析所有截图
       │
       └─ 超时自动提交（可配置）
               │
               ▼
        分析完成后保存为一条记忆
```

## 4. 设置修改流程

### 4.1 打开设置

```
菜单栏 → 文件 → 设置
       │
       ▼
创建 SettingsDialog 实例
       │
       ▼
传递依赖：
├── settings_manager
├── keyboard_manager
├── capture_manager
├── task_queue
└── ai_client (可选)
       │
       ▼
显示模态对话框
```

### 4.2 按钮行为矩阵

四个按钮的精确行为：

| 按钮 | 无变更 | 有变更无冲突 | 有变更且有冲突 |
|------|--------|-------------|----------------|
| **保存** | 静默无操作 | 持久化到磁盘，对话框保持打开 | 同左（当前无已知冲突） |
| **保存并应用** | 静默关闭对话框 | 持久化 + 运行时应用 + 关闭 | 弹出冲突对话框，确认后继续并关闭 |
| **取消** | 始终关闭，无提示（无脏检查） | 同左 | 同左 |
| **恢复默认** | 弹出确认对话框后重置 | 同左 | 同左 |

验证流程（`_validate_input()`）在保存/应用前执行：快捷键格式、API Key 空白检查、数值范围检查。

## 5. 错误处理流程

```
服务层发生错误
       │
       ▼
发射 error_occurred 信号
       │
       ▼
MainWindow._on_error()
       │
       ▼
状态栏显示："错误: {message}"
```

## 6. 快捷键流程

```
用户按下全局快捷键
       │
       ▼
keyboard_manager 捕获
       │
       ▼
发射 screenshot_requested 信号
       │
       ▼
MainWindow._on_screenshot()
       │
       ▼
执行截图逻辑
```

**快捷键配置**: 在 SettingsDialog 中修改后，通过 `_keyboard_manager.reload_hotkeys()` 热更新，无需重启应用。

## 8. 窗口关闭行为

```
用户点击窗口关闭按钮 (×)
       │
       ▼
closeEvent()
       │
       ▼
有系统托盘？
   ├── 是 → 隐藏窗口 + 显示托盘提示气泡
   └── 否 → 调用 _on_quit() 完全退出
       │
       ▼
_on_quit():
├── 隐藏托盘图标
├── 停止全局快捷键监听
├── 关闭任务队列
├── 关闭截图管理器
└── 退出 QApplication
```
