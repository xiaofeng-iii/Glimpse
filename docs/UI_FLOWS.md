# Glimpse 用户交互流程

## 1. 搜索流程

### 1.1 标准搜索（综合结果）

```
┌─────────────┐
│ 输入搜索词   │
└──────┬──────┘
       │ textChanged (300ms 防抖)
       ▼
┌─────────────┐
│ _do_search() │
└──────┬──────┘
       │ source_filter = "all"
       ▼
┌─────────────┐
│ SearchService│
│ .search()    │
└──────┬──────┘
       │ 同时执行：
       │ 1. SQLite FTS5 文本搜索
       │ 2. ChromaDB 向量搜索
       ▼
┌─────────────┐
│ RRF 融合排名 │
└──────┬──────┘
       │ 合并 + 去重
       ▼
┌─────────────┐
│ 标注来源标签 │
└──────┬──────┘
       │ OCR / 语义
       ▼
┌─────────────┐
│ _update_memory_list()
│ 渲染 [OCR][语义] 前缀
└─────────────┘
```

### 1.2 来源过滤搜索

用户在下拉框选择不同来源时的行为差异：

| 选择 | 执行方法 | 搜索结果 | 显示前缀 |
|------|----------|----------|----------|
| `综合结果` | `_search_hybrid()` | 文本+向量融合 | `[OCR]` / `[语义]` / 两者 |
| `仅看 OCR` | `_search_text()` | 仅文本匹配 | `[OCR]` |
| `仅看语义` | `_search_vector()` | 仅向量匹配 | `[语义]` |

**注意**: 切换下拉框会立即触发 `_do_search()`，无需重新输入搜索词。

## 2. 截图流程

### 2.1 单张截图

```
用户触发截图（按钮/快捷键）
       │
       ▼
隐藏主窗口
       │
       ▼
执行截图（250ms 延迟）
       │
       ▼
显示主窗口
       │
       ▼
发送信号 → MemoryService.create_memory_async()
       │
       ▼
状态栏显示进度："正在分析..."
       │
       ▼
完成回调 → 状态栏："记忆已保存 xxx"
       │
       ▼
发射 memory_saved 信号 → 刷新列表
```

### 2.2 集群截图

```
启用集群模式（SettingsDialog）
       │
       ▼
第一次截图
       │
       ▼
进入 COLLECTING 状态
       │
       ▼
显示"立即提交"和"取消"按钮
       │
       ├─ 用户继续截图 → 追加到缓冲区
       │   状态栏："已收集 N/M 张截图"
       │
       ├─ 用户点击"立即提交" → 分析所有截图
       │
       └─ 超时自动提交（可配置）
               │
               ▼
        分析完成后保存为一条记忆
```

## 4. 设置修改流程

### 4.1 打开设置

```
菜单栏 → 文件 → 设置
       │
       ▼
创建 SettingsDialog 实例
       │
       ▼
传递依赖：
├── settings_manager
├── keyboard_manager
├── capture_manager
├── task_queue
└── ai_client (可选)
       │
       ▼
显示模态对话框
```

### 4.2 修改并保存

```
用户在选项卡中修改配置
       │
       ▼
点击"确定"或"应用"
       │
       ▼
_collect_settings_from_ui()
       │
       ▼
_validate_input()
       │   ├── 快捷键格式验证
       │   ├── URL 格式验证
       │   └── 数值范围验证
       ▼
_apply_settings()
       │   ├── 更新截图设置
       │   ├── 重载快捷键
       │   ├── 配置 AI 客户端
       │   └── 保存到文件
       ▼
成功 → accept() 关闭对话框
失败 → 显示降级服务提示，不关闭
```

## 5. 错误处理流程

```
服务层发生错误
       │
       ▼
发射 error_occurred 信号
       │
       ▼
MainWindow._on_error()
       │
       ▼
状态栏显示："错误: {message}"
```

---

## 6. 快捷键流程

```
用户按下全局快捷键
       │
       ▼
keyboard_manager 捕获
       │
       ▼
发射 screenshot_requested 信号
       │
       ▼
MainWindow._on_screenshot()
       │
       ▼
执行截图逻辑
```

**快捷键配置**: 在 SettingsDialog 中修改后，通过 `_keyboard_manager.reload_hotkeys()` 热更新，无需重启应用。

## 7. 状态栏消息流

| 场景 | 状态栏文本 |
|------|-----------|
| 就绪 | "就绪" |
| 开始截图 | "正在截图..." |
| 截图过快 | "截图频率过快，请稍候..." |
| 分析中 | "截图完成，正在分析..." |
| 保存成功 | "记忆已保存: {id[:8]}..." |
| 保存失败 | "记忆创建失败" |
| 集群收集 | "已收集 {count}/{max} 张截图" |
| 集群倒计时 | "已收集 N/M 张截图 ({seconds}s后自动提交)" |
| 错误 | "错误: {message}" |

## 8. 窗口关闭行为

```
用户点击窗口关闭按钮 (×)
       │
       ▼
closeEvent()
       │
       ▼
有系统托盘？
   ├── 是 → 隐藏窗口 + 显示托盘提示气泡
   └── 否 → 调用 _on_quit() 完全退出
       │
       ▼
_on_quit():
├── 隐藏托盘图标
├── 停止全局快捷键监听
├── 关闭任务队列
├── 关闭截图管理器
└── 退出 QApplication
```
